(7) 触发器 TRIGGER

功能:

  1. 完成比约束更复杂的数据约束
  2. 检查操作是否允许
  3. 修改其他数据表中的数据
  4. 调用更多的存储过程
  5. 返回自定义的错误信息
  6. 防止数据表结构更改或数据表被删除

实验6是触发器专题https://sterne012.github.io/2021/11/19/DataBase6/

分类:

  1. DML触发器:响应数据操作语言(DML)事件时会自动执行的存储过程 包括INSERT UPDATE DELETE
    1. AFTER触发器:记录已经改变完之后才被激活执行,用于记录变更后的处理或检查
    2. INSTEAD OF触发器:取代原本的操作,不执行原来语句中的INSERT UPDATE DELETE操作,而是执行触发器本身定义的操作
  2. DDL触发器(ddl你不要过来啊):响应数据定义语言(DDL)即CREATE ALTER DROP
  3. 登录触发器:用户登录时触发,关注安全检查类问题

注意:

​ 触发器只能用于表和视图,同时只能建在一个表/视图上

​ 可以引用当前数据库以外的对象,但只能在当前数据库创建触发器

​ 不能在临时表和系统表上建触发器,但是可以引用临时表

​ TRUNCATE TABLE 不会激活DELETE 触发器(快速清空表中所有数据 TRUNCATE TABLE FROM SC)

inserted&deleted表:

触发器开始执行时产生,执行后消失,建在服务器内存中,只读,不能修改

inserted表:存储INSERT操作中新插入的数据&UPDATE操作中更新后的数据

deleted表:存储DELETE操作删除的数据&UPDATE操作中更新前的数据

事务:

触发器与引发触发器执行的操作共同构成事务,事务是一个整体,执行则都执行,不执行则都不执行

在AFTER触发器中使用ROLLBACK回滚整个事务

一般语法:BEGIN TRANSACTION………..ROLLBACK

(9) 存储过程 PROCDURE

优点:

  1. 预编译,减少编译语句所花的时间
  2. 存储过程内可以引用其他存储过程,简化复杂语句
  3. 减少网络通信流量
  4. 执行速度快
  5. 作为安全机制,用户不直接访问数据库,而是通过存储过程获得数据

语法:

CREATE PROCDURE <存储过程名>+number(可以过程名后加数字进行分组) + 参数名 数据类型…(=default 加默认值)(+OUTPUT 结果用输出参数返回调用者)AS…

若参数有默认值，则执行时可以不为有默认值的参数提供值

(10) 游标 CURSOR

使用select语句可以返回所有满足条件的行,但使用一行或者一部分行时,使用游标,游标支持对结果集中每一行进行相同/不同的操作

游标可以完成对多个update/insert语句的操作

DECLARE 游标名 CURSOR-FOR-SELECT…-FOR(READ ONLY/UPDATE)-

OPEN-

FETCH(NEXT/…)-FROM游标名-INTO变量-

WHILE…-

CLOSE-DEALLOCATE

DELCARE csr INSENSITIVE/SCROLL

INSENSITIVE

将游标提取出的数据放在一临时表中，对游标的读取操作均由临时表应答。因此对基本表的修改不影响游标提取的数据，也不能用游标更新基本表。不使用insensitive则可以对基本表进行更新

  1. SELECT中使用DISTINCT/GROUP BY/HAVING UNION
  2. 使用外连接
  3. 选取的表没有索引
  4. 将实数值当作选取的列

SCROLL

表明所有提取操作（FIRST/LAST/PRIOR/NEXT/RELATIVE/ABSOLUTE）都可用，不使用SCROLL则只能使用NEXT

CLOSE

把游标关闭，此时若没有insensitive声明，可以对基本表进行修改，游标中数据可能发生改变

再次使用OPEN可以再打开游标

更新、删除元组

前提是游标可更新

UPDATE表-SET…-WHERE CURRENT OF 游标

DELETE FROM表-WHERE CURRENT OF 游标

2. DROP 删除

(1) 模式 SCHEMA

DROP SCHEMA <模式名> <CASCADE|RESTRICT>

​ CASCADE级联删除：删除模式的同时把该模式中所有的数据库对象删除；

​ RESTRICT限制删除：当模式中没有任何下属对象的时候才能执行

​ DROP SCHEMA S-T CASCADE

(2) 表 TABLE

DROP TABLE <表名> <CASCADE|RESTRICT>

​ 删除表，表中数据、表上建立的索引、视图VIEW、触发器都将被删除

​ 使用RESTRICT，当表上有VIEW时，可以避免惨剧发生

​ DROP TABLE SC CASCADE
